# 觅食 · 大学城美食地图
### 产品介绍 & 设计思路说明

---

## 一、我们在解决什么问题

大学城是一个高度密集的消费场景：方圆 2 公里内聚集着数千名学生，每天三顿饭都要做决策，但他们面对的信息环境极其混乱。

大众点评的评论是面向大众的，不懂学生的钱包和口味；微信群里的推荐转瞬即逝，搜不到也存不了；学校 BBS 的内容又老又散，没有人维护。

更关键的是一个被所有人忽视的高频痛点——**今天中午吃什么？** 这个问题每天困扰着 90% 的学生，却没有任何一款 App 认真对待它。

**觅食** 就是为这个场景而生的。

---

## 二、产品定位

> **专为大学城设计的校园美食发现平台**
> 帮学生用最低的决策成本，找到最适合当下场景的一餐。

我们不想做一个"小型大众点评"。觅食的核心价值在于三个字：**懂你的场景**。

同样是吃饭，备考期间需要的是快、便宜、不打扰；约会时需要的是氛围、拍照好看；团建时需要的是能坐很多人、适合喝酒聊天。没有哪款通用工具能把这些场景理解得这么细，但我们可以。

---

## 三、核心功能与设计思路

### 3.1 探索页：场景优先，而不是品类优先

传统餐饮 App 的筛选逻辑是"火锅 / 快餐 / 日料"，这是从商家视角出发的分类。

我们把筛选逻辑重新设计成**场景维度**——减脂、约会、团建、备考、深夜、省钱。用户不需要知道自己想吃什么类型的菜，只需要告诉 App 他现在处于什么状态。这一个改变，让信息架构从"我卖什么"变成了"你需要什么"。

分类 Chip 和场景 Chip 可以组合过滤，学生甚至可以同时选择"北区商圈 + 约会场景"，精准命中最合适的选项，而不是在几十条结果里自己判断。

### 3.2 "今天吃什么"转盘：把决策焦虑变成游戏

选择困难症是学生群体里普遍存在的现象，尤其是在三五个人一起讨论吃什么的时候，讨论本身比吃饭花的时间更长。

转盘功能的核心逻辑不是随机，而是**把一个有压力的决策变成一个有仪式感的小游戏**。当结果由"算法"给出，群体里没有人需要负责，矛盾就消解了。心理学上这叫责任分散——我们只是把它做成了一个产品功能。

从设计角度，转盘的结果会直接给出距离、排队时间、一键导航，把"抽到了"和"去吃"之间的摩擦降到最低。

### 3.3 实时排队系统：信息的时效性是核心竞争力

普通点评 App 的排队信息是用户事后写的，滞后且不准。我们设计了一个**实时排队波动模拟机制**（未来对接商家端实时更新），每 8 秒刷新一次队列状态，并用颜色 + 动效做区分：

- 🟢 绿色无动效 = 无需排队，现在去最合适
- 🟡 黄色脉冲 = 中等等待，可以考虑
- 🔴 红色脉冲 = 队很长，谨慎决策

这个设计的逻辑是：**颜色传达情绪，动效传达紧迫性**。用户扫一眼列表，不需要阅读文字，状态就已经传达到了。

### 3.4 地图页：范围感知 + Pin 聚焦

地图模式对于"我在哪里、周围有什么"的用户来说，比列表更直觉。

我们设计了一个范围滑块，让用户用"距离"来筛选，而不是用行政区划。一个在宿舍楼附近的学生，他关心的是"300 米以内有什么"，而不是"北区有什么"。滑块动态调整覆盖圆圈，哪些 Pin 亮起、哪些变灰，一目了然。

底部悬浮卡片固定展示当前选中餐厅的关键信息（距离、排队状态、优惠），点击直接进详情，减少了从"地图发现"到"决定去吃"之间的跳转层级。

### 3.5 社区页：UGC 的信任杠杆

我们认为校园美食场景里，**同学的推荐永远比算法推荐可信**。一个认识的人发了"这家踩雷了"，比 3.2 分的评分更有说服力。

社区模块的设计重点有两个：

第一是"踩雷黑榜"——这是一个反向筛选工具。不告诉你去哪吃，而是告诉你不要去哪。在一个新生开学、信息不对称严重的环境里，避雷的价值不亚于推荐。这个功能本身就是很强的话题性入口，天然适合传播。

第二是帖子与餐厅数据打通——用户的社区帖子可以直接关联具体餐厅，点击位置标签就能跳转到餐厅详情页。UGC 内容不是孤立的，它成了餐厅页面的有机组成部分，形成内容 → 决策的闭环。

### 3.6 个人中心：身份认同 + 福利驱动留存

学生用户的留存动机很简单：**占到便宜，同时感觉自己属于一个圈子**。

等级体系（探索家、美食侦探等）提供的是圈子感。领券中心提供的是实际利益。学生邮箱认证是一个权益门槛，认证后解锁更多商户折扣，这个设计让"真实学生"的身份变成一种资产，同时对平台做了用户质量的初步筛选。

## 四、目标用户与使用路径

**核心用户：** 大学城内在校学生，18–25 岁，对价格敏感，对体验有要求，高度依赖手机做消费决策。

**典型使用路径（午饭场景）：**

```
打开 App → 选择"速食"场景 → 
看到附近排队少的餐厅 → 进详情确认菜品 → 
导航前往 → 吃完发帖 → 形成 UGC
```

**另一条路径（纠结场景）：**

```
打开 App → 点击"今天吃什么" → 
转盘抽到结果 → 看排队状态 → 
一键导航前往
```

两条路径都在 3 步以内完成核心决策，这是我们在信息架构上的基本原则：**从打开 App 到做出决定，不超过 3 次点击**。

---

## 五、商业化路径（阶段规划）

**阶段一（冷启动）：** 免费入驻，以内容质量和用户密度为核心指标，在 1–2 所高校跑通模型。

**阶段二（变现探索）：** 向商家提供排队数据可视化报告、竞品分析工具（SaaS 订阅）；推出学生专属优惠券分发系统（按核销抽佣）。

**阶段三（规模扩张）：** 在高校聚集的城市复制，形成"大学城美食数据网络"，向本地生活平台提供数据授权或流量合作。
